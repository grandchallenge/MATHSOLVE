# TROPIC-GROEBNER Orchestration

## Purpose

This lane tells MATHSOLVE when to use tropical degeneration as a route-selection tactic for polynomial obligations.

The doctrine is:

> MATHSOLVE chooses the degeneration. MATHCERT checks the initial-ideal claim.

TROPIC-GROEBNER is not a symbolic algebra backend. It is the tactical layer that turns support geometry and coefficient valuation data into bounded weighted Gröbner computations, then records whether each weight was retained or rejected.

## When to invoke the lane

A work package may invoke TROPIC-GROEBNER when the local proof or computation obligation involves:

- sparse polynomial or Laurent-polynomial systems;
- coefficient valuations or degeneration limits;
- toric, binomial, or Newton-polytope structure;
- a need to decide whether a sampled weight belongs to a tropical variety;
- a Gröbner computation where the term-order choice is itself a strategic question.

Do not invoke the lane merely because a problem has polynomials. Ordinary ideal membership, radical membership, resultants, quotient algebras, and direct Gröbner reduction remain the first route when support geometry adds no useful structure.

## Outputs

A successful invocation should produce:

- a route decision report;
- a weight ledger with accepted and rejected sampled weights;
- exact initial-form or initial-ideal records;
- monomial witnesses for rejected weights;
- a MATHCERT handoff when the local statement is worth replaying.

## Route decision states

| State | Meaning |
| --- | --- |
| `candidate_weight` | Weight generated by support, valuation, or router heuristic. |
| `weight_retained` | Initial ideal is recorded as monomial-free under the declared check. |
| `weight_rejected` | A monomial witness was found in the initial ideal. |
| `route_selected` | The retained degeneration improves the local obligation under budget. |
| `route_switched` | Another exact route is smaller or clearer. |
| `ready_for_mathcert` | Certificate payload is stable enough for replay. |

Only a returned MATHCERT result may change a mathematical claim to checked or certified.

## First fixture campaign

Use `TROPIC-GROEBNER-001` from MATH-PROGRAMME:

```text
I = <x + y + 1> in QQ[x, y]
valuation = trivial
weights = (0,0), (1,0), (-1,0), (1,2)
```

Expected decisions:

```text
(0,0)   -> in_w(f) = x + y + 1 -> retained
(1,0)   -> in_w(f) = y + 1     -> retained
(-1,0)  -> in_w(f) = x         -> rejected, witness x
(1,2)   -> in_w(f) = 1         -> rejected, witness 1
```

## Trust boundary

MATHSOLVE may retain a degeneration as useful route evidence. It may not claim that a sampled tropical membership test enumerates the tropical variety, solves the original system, or proves a source theorem.
